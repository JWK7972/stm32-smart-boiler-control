# Smart Boiler Control System

STM32와 RTOS를 이용해 만든 스마트 보일러 제어 시스템 프로젝트입니다.  
입력, 제어, 표시 기능을 나누어 구성했고, 상태에 따라 LCD와 LED에 동작 결과를 표시하도록 구현했습니다.

## 1. 프로젝트 개요

이 프로젝트는 보일러의 온도 설정, 동작 모드, 전원 상태를 입력받아  
설정 조건에 맞게 시스템이 동작하도록 만든 임베디드 제어 프로젝트입니다.

기능을 하나의 반복문에 모두 넣기보다,  
입력 처리, 제어 판단, LCD 표시, LED 표시를 분리해서  
조금 더 구조적으로 구현해보는 것을 목표로 했습니다.

## 2. 개발 환경

- MCU : STM32
- Language : C
- IDE : STM32CubeIDE
- RTOS : uC/OS-II
- Display : 문자 LCD
- Output : LED
- Input : 버튼, 온도 설정 입력

## 3. 주요 기능

- 전원 ON/OFF 입력 처리
- 수동/자동 모드 전환
- 목표 온도 설정
- 현재 상태에 따른 제어 동작 수행
- LCD를 통한 상태 표시
- LED를 통한 동작 상태 표시
- RTOS Task 기반 구조 분리

## 4. 구성 방식

기능을 다음과 같이 나누어 구현했습니다.

- 입력 Task : 버튼이나 설정값 입력 처리
- 제어 Task : 현재 상태와 설정값을 바탕으로 동작 판단
- LCD Task : 표시 내용 갱신
- LED Task : 상태에 맞는 LED 출력

각 기능을 분리하면서,  
공유 데이터 전달에는 flag와 semaphore를 사용했습니다.

## 5. 구현하면서 중점적으로 본 부분

이 프로젝트에서는 단순히 입력에 따라 출력만 바꾸기보다,  
상태가 바뀔 때 어떤 정보가 다른 기능으로 전달되어야 하는지를 정리하는 데 중점을 두었습니다.

특히 표시 장치는 동시에 접근하면 꼬일 수 있다고 생각해서  
세마포어를 이용해 자원 접근을 나누었고,  
전원 OFF 상태에서는 제어 동작이 불필요하게 이어지지 않도록 상태 흐름을 정리했습니다.

## 6. 맡은 역할

- RTOS 기반 Task 구조 작성
- 입력 / 제어 / LCD / LED 기능 분리
- 상태 전달 방식 정리
- LCD 출력 내용 구성
- LED 상태 표시 로직 구현
- 동작 테스트 및 로직 수정

## 7. 배운 점

이 프로젝트를 하면서  
기능이 많아질수록 한 곳에 모두 넣는 구조보다  
역할별로 분리해서 관리하는 것이 훨씬 보기 쉽고 수정하기 편하다는 점을 느꼈습니다.

또한 RTOS를 사용하면서  
주기와 우선순위, 공유 자원 접근 방식이 중요하다는 점도 직접 확인할 수 있었습니다.

## 8. 실행 결과

- 전원 및 모드 입력에 따라 정상 동작
- 설정값에 따른 상태 변화 확인 가능
- LCD와 LED를 통해 현재 상태 표시 가능
- Task 분리를 통해 기능별 동작 흐름 확인 가능

## 9. 파일 구성

- `main.c` : 전체 실행 구조
- `Task 관련 코드` : 입력, 제어, LCD, LED
- `LCD 관련 함수`
- `LED 관련 함수`
- `상태 전달 관련 코드`
